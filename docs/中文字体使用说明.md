# 中文字体使用说明

本文说明如何为本项目准备和使用简体中文字体，包括：

- 在哪里获取 `ttf/otf` 字体源文件
- 如何把字体转换成设备可用的 `.cpfont`
- 如何把转换后的字体放到 SD 卡
- 如何在设备里启用中文字体

## 1. 先了解格式要求

设备不能直接使用 `ttf` 或 `otf`。

本项目读取的是专用的 `.cpfont` 文件，字体目录规则如下：

```text
/.fonts/<Family>/
```

也兼容：

```text
/fonts/<Family>/
```

例如：

```text
/.fonts/SourceHanSansSC/SourceHanSansSC_12.cpfont
/.fonts/SourceHanSansSC/SourceHanSansSC_14.cpfont
/.fonts/SourceHanSansSC/SourceHanSansSC_16.cpfont
/.fonts/SourceHanSansSC/SourceHanSansSC_18.cpfont
```

推荐优先使用隐藏目录 `/.fonts`。

## 2. 获取中文字体源文件

推荐使用官方开源中文字体：

- 思源黑体 `Source Han Sans SC`
  - 发布页：https://github.com/adobe-fonts/source-han-sans/releases
  - 项目页：https://github.com/adobe-fonts/source-han-sans
- Noto CJK
  - 项目页：https://github.com/notofonts/noto-cjk
  - Noto 官方使用页：https://notofonts.github.io/noto-docs/website/use/

建议优先使用简体中文无衬线字体：

- `SourceHanSansSC-Regular.otf`
- `SourceHanSansSC-Bold.otf`

如果你要做衬线版，也可以换成：

- `Noto Serif CJK SC`
- `Source Han Serif SC`

## 3. 准备 Python 依赖

本项目自带字体转换脚本：

```text
lib/EpdFont/scripts/fontconvert_sdcard.py
```

依赖清单在：

```text
lib/EpdFont/scripts/requirements.txt
```

在仓库根目录运行：

```powershell
python -m pip install --user -r lib\EpdFont\scripts\requirements.txt
```

## 4. 转换字体

### 4.1 为什么不能只用 `cjk`

如果只使用：

```text
--intervals cjk
```

那么通常只会包含中文和 CJK 相关字符，不一定包含英文、数字和常见西文标点。这样中文文件名能显示，但英文命名文件可能会缺字。

因此推荐使用：

```text
--intervals latin-ext,cjk
```

这样会同时包含：

- 简体中文
- 英文大小写
- 数字
- 常见西文标点

### 4.2 推荐生成的字号

建议生成：

```text
12,14,16,18
```

原因：

- 阅读正文会使用当前阅读字号
- 当前固件还会额外固定使用 `12pt` 来显示书名、作者、文件名、章节名等用户内容文本

### 4.3 以 `SourceHanSansSC` 为例的转换命令

如果你的字体源文件位于：

```text
D:\dgx\source\Font\OTF\SimplifiedChinese
```

可在仓库根目录运行：

```powershell
python lib\EpdFont\scripts\fontconvert_sdcard.py `
  --intervals latin-ext,cjk `
  --sizes 12,14,16,18 `
  --name SourceHanSansSC `
  --regular "D:\dgx\source\Font\OTF\SimplifiedChinese\SourceHanSansSC-Regular.otf" `
  --bold "D:\dgx\source\Font\OTF\SimplifiedChinese\SourceHanSansSC-Bold.otf" `
  --output-dir "Other\generated-fonts\SourceHanSansSC"
```

说明：

- `--name SourceHanSansSC`
  - 指定字体家族名称
- `--regular`
  - 指定常规字重字体文件
- `--bold`
  - 指定粗体字重字体文件
- `--sizes 12,14,16,18`
  - 一次性生成多个字号
- `--intervals latin-ext,cjk`
  - 同时包含英文和中文

### 4.4 如果没有 italic 和 bolditalic

没关系。

只提供 `regular + bold` 也可以正常使用。当前固件会对缺失的样式做回退处理。

## 5. 转换完成后会得到什么

以上命令会输出到：

```text
Other/generated-fonts/SourceHanSansSC/
```

生成文件通常为：

- `SourceHanSansSC_12.cpfont`
- `SourceHanSansSC_14.cpfont`
- `SourceHanSansSC_16.cpfont`
- `SourceHanSansSC_18.cpfont`

本仓库当前已经生成好了一套可直接使用的文件，位置在：

```text
Other/generated-fonts/SourceHanSansSC/
```

## 6. 把字体放到 SD 卡

将生成好的 4 个 `.cpfont` 文件复制到 SD 卡：

```text
/.fonts/SourceHanSansSC/
```

完整示例：

```text
/.fonts/SourceHanSansSC/SourceHanSansSC_12.cpfont
/.fonts/SourceHanSansSC/SourceHanSansSC_14.cpfont
/.fonts/SourceHanSansSC/SourceHanSansSC_16.cpfont
/.fonts/SourceHanSansSC/SourceHanSansSC_18.cpfont
```

如果目录不存在，就手动创建：

- `/.fonts`
- `/.fonts/SourceHanSansSC`

## 7. 也可以通过网页上传

如果不想手动把文件拷到 SD 卡，也可以：

1. 让设备连上 Wi-Fi
2. 打开设备网页管理界面
3. 进入 `/fonts`
4. 上传同一家族下的 `.cpfont` 文件

注意：

- 网页上传接受的是 `.cpfont`
- 不是 `ttf`
- 也不是 `otf`

## 8. 在设备里启用中文字体

把字体放入 SD 卡后，在设备上进入：

```text
Settings -> Reader -> Reader Font Family
```

然后选择：

```text
SourceHanSansSC
```

字体管理入口在：

```text
Settings -> Reader -> Manage Fonts
```

## 9. 推荐使用顺序

建议按下面顺序操作：

1. 获取 `SourceHanSansSC-Regular.otf` 和 `SourceHanSansSC-Bold.otf`
2. 安装 Python 依赖
3. 运行转换命令生成 `12/14/16/18` 四个 `.cpfont`
4. 复制到 `/.fonts/SourceHanSansSC/`
5. 在设备中选择 `Reader Font Family -> SourceHanSansSC`
6. 重启设备，或重新进入一次字体选择页确认加载生效

## 10. 常见问题

### 10.1 中文能显示，但英文文件名不显示

原因通常是转换时只用了：

```text
--intervals cjk
```

解决办法：

改为重新生成：

```text
--intervals latin-ext,cjk
```

### 10.2 直接把 `ttf/otf` 放到 SD 卡里，设备看不到

这是正常的。

设备只识别 `.cpfont`，必须先转换。

### 10.3 只生成了 `14/16/18`，列表页书名仍有问题

建议补上 `12pt`，因为当前固件会额外使用固定 `12pt` 显示用户内容文本。

### 10.4 字体文件很大

中文字体本身覆盖字符多，生成后的 `.cpfont` 体积明显大于拉丁字体，这是正常现象。

- `D:\dgx\source\Font\OTF\SimplifiedChinese\SourceHanSansSC-Bold.otf`
